# PTF Április — Interface Quality Audit

**Surface audited:** login, waiting, closed, success (two implementations), in-round game shell, chat, hint panel, answer entry, email modal, footer, global theme.
**Report date:** 2026-04-24 (competition day)

---

## Anti-Patterns Verdict

**FAIL.** This interface reads as AI-generated within five seconds. The fingerprints are all over it:

- **"Matrix rain" canvas background** on login + waiting (`components/matrix-bg.tsx`) — the single most overused "AI hacker aesthetic" trope from 2023–2025. Bonus points: it renders in **blue `#3b82f6`** while the rest of the app is green `#00ff88`, so the one visual flourish doesn't even match the palette.
- **Dark mode + neon green accents** on a pure-dark surface (`#0a0a0f`) — the "cool without requiring actual design decisions" default the frontend-design skill explicitly calls out.
- **Purple → blue gradient buttons** (`bg-gradient-to-r from-purple-600 to-blue-600`) in the email modal and the stale `/success` page — the archetypal AI-slop gradient. Directly listed as a **DON'T** in the skill.
- **Big-number-with-small-label "hero metric" cards** repeated 4× on waiting (countdown), closed, and success pages — the exact "hero metric layout template" the skill warns against.
- **Icon-in-rounded-colored-box above every heading** (`<Shield>` in a `w-20 h-20 rounded-2xl bg-[#00ff88]/10 border-2`) on login, waiting, closed, success — the "large icons with rounded corners above every heading" that makes sites look templated.
- **Geist + Geist Mono** — listed as generic system-font-adjacent defaults in the skill. The display font ≡ the body font ≡ what ships with create-next-app. There is no distinctive voice.
- **`font-mono` used as shorthand for "technical"** in the countdown, header timer, passcode field, and answer field — "lazy shorthand for developer vibes."
- **Identical card grids** — the same `bg-black/70 border border-[#00ff88]/30 rounded-xl p-4` container pattern appears 14+ times (waiting countdown, closed metrics, success metrics, per-round breakdown, username panel).
- **Glassmorphism** — `backdrop-blur-sm` used decoratively on login card, waiting info panel, video skip button, footer.
- **Everything centered** — every marketing-style page (login, waiting, closed, success) is a vertically centered single column with centered text.

**The "could a designer have made this?" test:** No. It looks exactly like a v0/Lovable dark-hacker-theme template with the brand color swapped to green. Zero aesthetic conviction.

- **Category:** Content / Theming
- **Issue:** The `/success` route (which `middleware.ts:75` redirects all solved users to) renders the *March* competition UI. - **Impact:** Every solver today downloads a certificate claiming they solved *March*. Reputational and user-trust damage.
- **Recommendation:** Delete `app/success/page.tsx` and re-point middleware to render `components/game/phase-success.tsx` (which is already updated for April but currently orphaned — it's only reachable via the `SUCCESS` game phase, which solved users never hit because middleware redirects them first). - **Suggested command:** `/harden` (root-cause this first, no design command will fix stale copy)

**C2. Missing focus-visible indicators on custom controls**
- **Location:** `app/login/page.tsx:97,111`, `components/chat/chat-input.tsx:55,57`, `components/round/answer-entry.tsx:76,88,91,110,123`, `components/game/phase-success.tsx:337,352,365,386`, `components/chat/clear-context-button.tsx:10`, `components/chat/try-door-button.tsx:36`
- **Category:** Accessibility
- **WCAG:** 2.4.7 Focus Visible (AA), 2.4.11 Focus Appearance (AAA)
- **Issue:** Every hand-rolled `<input>` / `<button>` uses `focus:outline-none` and changes only a 1px border color from `white/10` → `[#00ff88]/50`. Keyboard users effectively cannot see which control is focused. The Radix UI components (`ui/button.tsx`, `ui/input.tsx`) do have `focus-visible:ring-[3px]` — but the game chat, hints, answer entry, login, and phase-success all bypass those primitives.
- **Impact:** Keyboard-only users (and participants using assistive tech during a 60-minute timed contest) can't see where they are.
- **Recommendation:** Add `focus-visible:outline-2 focus-visible:outline-offset-2 focus-visible:outline-[#00ff88]` (or `ring` equivalent) to every custom control. Use the `ui/*` primitives for new code.
- **Suggested command:** `/harden`

**C3. Matrix canvas draws in wrong brand color**
- **Location:** `components/matrix-bg.tsx:51,57`
- **Category:** Theming
- **Issue:** The falling-characters background is hard-coded to `#3b82f6` (Tailwind blue-500), but every other surface uses `#00ff88` (neon green). The login page and waiting page show blue rain behind green UI.
- **Impact:** Immediate brand-cohesion break on the two pages users see first.
- **Recommendation:** Either change to `#00ff88` to match, or (better, per the anti-patterns) **remove the matrix background entirely** — it's AI-slop shorthand for "hacker" that adds nothing.
- **Suggested command:** `/normalize` (if keeping), `/distill` (to remove)

**H5. `Lock`-icon + countdown disclosure text has 9-15% contrast**
- **Location:** `components/round/hint-panel.tsx:66–72` (`text-white/20`), `:91–97` (`text-white/40`), `app/login/page.tsx:129` (`text-white/20`)
- **Category:** Accessibility
- **WCAG:** 1.4.3 Contrast (Minimum) AA
- **Issue:** `text-white/20` on `#0a0a0f` yields ~1.5:1 contrast; `text-white/40` yields ~3.2:1. Body copy fails AA (needs 4.5:1). These aren't decorative — they contain live countdown information ("Tipp 2 — 4:37").
- **Impact:** Users with low vision or on glare-affected screens cannot read the countdown that gates their ability to unlock hints.
- **Recommendation:** Minimum `text-white/60` for informational content; `text-white/80` for interactive affordances.

- **Issue:** The April competition has no coupon mechanic (CLAUDE.md confirms the April flow extracts a secret code to "escape the building", not a discount coupon). This pill appears regardless and is factually wrong for April.
- **Impact:** Confuses users who don't remember a coupon being part of the challenge.

**H7. Monospace abuse for "technical" vibes**
- **Location:** `app/waiting/page.tsx:90,96,102,108` (countdown), `app/login/page.tsx:97` (invite code entry), `components/layout/header-bar.tsx:57` (timer), `components/round/answer-entry.tsx:121` (safe code)
- **Category:** Typography / Anti-pattern
- **Issue:** Mono is being used as shorthand for "digital/technical" — specifically the DON'T in the skill: "monospace typography as lazy shorthand for 'technical/developer' vibes." Countdown digits don't need mono (they need tabular-nums, which the sans can do with `font-variant-numeric`).
- **Impact:** Generic "terminal" look, not the editorial look the heist theme could support.
- **Recommendation:** Use `tabular-nums` from Geist (or better, a display-serif with lining numerals for headings). Restrict mono to the safe code entry and the actual typed-response preview, where mechanical character-by-character reading matters.

## Patterns & Systemic Issues

1. **The "card" is the only layout unit.** Twenty-plus occurrences of the same `bg-black/70 border-[#00ff88]/xx rounded-xl p-N` container. There is no rhythm, no asymmetry, no grid-breaking emphasis. Every page is a vertical stack of identical boxes.
2. **The design is forked, not redesigned.** The success page, email modal, and certificate PDF generator still carry March's visual language (light theme, blue/purple gradients, "Márciusi kihívás" body text). The dark-matrix-green reskin was applied to the in-round experience and the login/waiting/closed pages but didn't reach the post-solve surface.
3. **Focus management is absent app-wide.** Every hand-rolled control drops the outline with `focus:outline-none` and relies on a weak border color change. The Radix `ui/*` primitives have good `focus-visible:ring` defaults but aren't used by the game shell.
4. **Low-opacity text carries information.** `text-white/20`, `text-white/30`, `text-white/40` are used for 28 informational elements (labels, countdowns, status). All fail WCAG AA.
5. **Monospace is overused as a signifier.** Timer, countdown, passcode entry, invite code, safe code — five different mono slots on a site that's mostly chat. The "terminal" aesthetic is the DON'T.
6. **Brand palette isn't enforced.** Green (`#00ff88`), blue (`#3b82f6` matrix), purple (modal), LinkedIn blue (`#0A66C2`), red (`red-400`, `red-500`), and Tailwind default blue/purple in the success page. No token discipline.

## Positive Findings

- **Radix primitives are in place.** `ui/button.tsx`, `ui/input.tsx`, `ui/dialog.tsx` have proper `focus-visible:ring`, `aria-invalid` hooks, and CVA variants — they're just not used by most game surfaces.
- **OKLCH color tokens are defined in `globals.css`.** The foundation for a coherent perceptual palette exists (even if nothing consumes it — every component uses arbitrary `#hex` values).
- **Phase-success (components/game/phase-success.tsx)** is the better-April-themed success page — it already has the heist voice ("Infiltráció sikeres", "Küldetés teljesítve"), per-round breakdown, and a green certificate. It's just unreachable. Re-route middleware to render it instead of the stale `/success` page.
- **Chat input has thoughtful ergonomics** — auto-resize with max-rows cap, enter-to-send with shift-enter newline, proper `disabled` handling during send.
- **Timer has sensible color stages** (green → yellow at 5min → red at 1min → bold-red at 0). The information design is right; only the assistive-tech story and contrast are weak.
- **Hint panel's unlock-over-time model** is a nice, purposeful progressive disclosure — exactly the kind of interaction the design skill endorses.
- **`matrix-bg.tsx` cleanup** is proper — returns cleanup from useEffect, cancels `requestAnimationFrame`, removes resize listener. It shouldn't exist, but the code is clean.
